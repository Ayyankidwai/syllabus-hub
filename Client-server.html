Client-Server Model

Client-server architecture is a computer network structure where multiple clients (remote processors) request and receive services from a centralized server (host computer). Client computers provide an interface for users to request services from the server and display the results returned. Servers await requests from clients and respond accordingly. Ideally, a server provides a standardized, transparent interface so that clients are unaware of the underlying system (hardware and software) providing the service. Clients are often situated at workstations or personal computers, while servers are located on more powerful machines elsewhere on the network. This computing model is effective when clients and servers have distinct tasks. For example, in hospital data processing, a client computer can enter patient information while the server manages the database where the information is stored. Many clients can access the server simultaneously, and a client computer can perform other tasks like sending emails. Unlike the old mainframe model, where a centralized mainframe performed all tasks for its "dumb" terminals, the client-server model treats client and server computers as independent devices.

How the Client-Server Model Works

Client: In digital terms, a Client is a computer (Host) capable of receiving information or using a service from service providers (Servers).
Server: A Server is a remote computer that provides information or access to services.

How the Browser Interacts with the Server

  • DNS Server looks up the address of the WEB Server.

Advantages of the Client-Server Model

  1. Centralized system with all data in a single place.
  2. Cost-efficient, with lower maintenance costs and data recovery options.
  3. The capacity of Clients and Servers can be adjusted separately.

Disadvantages of the Client-Server Model

  1. Clients are prone to viruses, Trojans, and worms if present in or uploaded to the Server.
  2. Servers are vulnerable to Denial of Service (DOS) attacks.
  3. Data packets may be spoofed or modified during transmission.

Conclusion

Client-server architecture consolidates resources on servers for greater control and security, allows for flexible client options, and relies on a robust network for scalability and efficiency. While there are cost implications, the client-server model remains fundamental and continues to evolve with trends such as cloud computing.
